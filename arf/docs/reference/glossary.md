# Glossary

**aggregator** — A Python script that combines one type of data from across task folders into a
single output. Supports filtering by category and/or task IDs.

**asset** — Any artifact produced by a task. Stored under `assets/<asset_type>/<asset_id>/` in the
producing task folder.

**asset type** — A category of asset such as `paper`, `dataset`, `library`, `model`, `answer`, or
`predictions`. Defined in `meta/asset_types/`.

**category** — A tag assigned to tasks and assets to enable filtering. Each category is a folder
under `meta/categories/`.

**correction** — A JSON overlay created by a downstream task that can update, replace, or delete
metrics, suggestions, or assets from previously completed tasks without modifying their folders.
Format defined in `arf/specifications/corrections_specification.md`.

**cost** — Third-party expenses incurred by a task: paid API calls, paid remote compute. Recorded in
`results/costs.json`.

**dependency** — A task ID that must be completed before another task can start. Declared in
`task.json`.

**diagnostic code** — A short identifier (e.g., `FD-E001`, `PA-W003`) emitted by a verificator to
flag a specific error or warning.

**intervention** — A file in `intervention/` describing a blocking issue that requires human action
(e.g., dataset access approval, budget override).

**log** — A structured record of a step, command, search, or session. Stored under `logs/` in the
task folder. Format defined in `arf/specifications/logs_specification.md`.

**metric** — A registered quantitative measurement declared in `meta/metrics/` and reported in a
task's `results/metrics.json`.

**news** — A daily project digest written to `news/$DATE.md` and `news/$DATE.json`. Summarizes the
day's completed tasks, costs, papers, and key findings. Produced by the `generate-daily-news` skill.

**overview** — Committed snapshots of aggregator outputs under `overview/`, used for human review
through GitHub. Regenerated by `arf/scripts/overview/materialize.py`.

**plan** — The execution plan for a task. Written to `plan/plan.md` during the planning stage.

**predictions** — An asset type holding model predictions produced by a task.

**skill** — A reusable agent workflow definition under `arf/skills/<slug>/SKILL.md`, exposed via
`.claude/skills/` and `.codex/skills/`.

**specification** — A document under `arf/specifications/` defining the required format for a file
or artifact class. Consumed by verificators and agents.

**step** — One unit of execution within a task, usually run by a dedicated subagent. Tracked in
`step_tracker.json`.

**step tracker** — The `step_tracker.json` file in a task folder. Lists all steps and their
execution state.

**suggestion** — A task candidate (hypothesis, dataset, library, etc.) proposed by a completed task.
Stored in `results/suggestions.json`.

**task** — An atomic unit of project work. Lives in its own folder `tasks/<task_id>/`, runs on its
own branch, and produces a single pull request.

**task ID** — The identifier of a task, in the format `t<NNNN>_<slug>` (lowercase `t`, 4-digit
number, underscore, slug). Must be a valid Python identifier.

**task type** — A classification of task such as `baseline-evaluation`, `experiment-run`, or
`literature-survey`. Defined in `meta/task_types/`.

**verificator** — A Python script that checks files against a specification and emits errors and
warnings identified by diagnostic codes.
